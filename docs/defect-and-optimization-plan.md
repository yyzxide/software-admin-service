# 缺陷和优化计划

## 已处理

- 操作日志从旁支能力并入 `admin-service` 主线。
- 新增 `operation_logs` 表，支持日志写入、列表、详情、选项和统计。
- 新增审核任务表和审核历史表。
- 新增审核接口：提交、分配、通过、驳回。
- 软件、版本、安装包和审核动作接入操作审计。
- 修复管理页面错误提示中不存在的 `make run-admin-service` 命令。
- 联调检查增加审核任务和操作日志接口。
- 重做静态管理台视觉和信息架构，补齐审核中心、操作日志页面。
- 上传链路增加单文件大小、业务安装包大小、文件后缀和包格式一致性校验。
- 登录支持 BCrypt 密码哈希、Token 会话版本和历史 SHA-256 哈希兼容升级。
- 增加安全和上传限制设计文档。
- 接入 springdoc-openapi，提供 Swagger UI 和 `/v3/api-docs`。
- 增加 `make smoke` 本地冒烟脚本。
- 增加演示清单和简历项目描述文档。
- 增加安装包分片上传、断点续传、合并校验、SHA256 完整性校验和可选 RSA 签名验签。
- 上传软件、新增版本、追加安装包支持消费已完成上传会话。
- 增加 RBAC 权限模型，支持管理员、角色、权限点、用户角色、角色权限和接口注解拦截。
- 增加过期上传会话定时清理，自动删除 `.chunks` 临时分片目录。
- 增加审核通过和上架前的安装包安全状态拦截。
- 增加 admin-service Dockerfile 和 Compose app profile，支持后端一键容器化启动。
- 测试覆盖持续补充，已覆盖上传、审核、权限、日志和安全拦截等核心路径。
- 管理台弱化裸 ID 输入，分类、标签、软件、版本、角色和权限都通过名称或编码选择。
- 抽离 `SoftwareAssembler`，把软件、版本、安装包响应 DTO 组装从核心业务 Service 中拆出。
- 抽离 `PackagePreparationService`，统一安装包规格校验、直传/分片会话互斥校验和 `AppPackageEntity` 构建，并补充直接单元测试。
- 修复空扫描请求默认安全、非法 Token 可能返回 500、配置管理员非本地兜底、审核分配覆盖和重复活动审核任务竞态。

## 当前缺陷

### 1. 前端仍是轻量验证台

当前 `admin-ui` 已覆盖软件、审核、操作日志、权限管理、分类、标签和联调检查，适合本地演示和接口验证。它仍然是纯静态页面，复杂交互、图表和权限菜单还没有做。

建议：后续只做必要增强，例如审核详情弹层、日志统计图和更完整的错误提示，不引入重型前端工程。

### 2. 上传治理仍需生产化

当前已经支持普通 Multipart、分片上传、断点续传、合并校验、SHA256 校验、可选 RSA 验签和过期临时分片清理，能覆盖研发阶段的大文件上传链路。但仍是本地文件存储，正式包孤儿文件清理还没有做成后台任务。

建议：增加正式包孤儿文件清理、失败补偿策略，并预留对象存储迁移接口。

### 3. 权限管理仍可生产化

当前已有 RBAC 表、默认角色权限数据、接口权限注解拦截、管理员账号管理、角色管理和角色授权页面。

建议：继续增加登录失败次数限制、密码复杂度策略、菜单级权限和按钮级权限隐藏。

### 4. API 文档仍可继续增强

当前已接入 springdoc-openapi 和 Swagger UI，可以在线查看和调试接口。后续如果希望更贴近国内 Java 后台项目习惯，可以替换或补充 Knife4j UI。

### 5. 部署链路仍需生产化

当前支持 Docker Compose app profile 一键启动 MySQL、Redis 和后端，但还没有生产级健康检查、镜像版本策略和部署说明。

建议：增加 admin-service 健康检查、镜像 tag 规范和生产部署文档。

## 优先级建议

1. 生产级密码策略、登录失败限制和菜单级权限隐藏。
2. 正式包孤儿文件清理和安装包安全扫描执行器。
3. Docker 生产部署说明。
4. 更完整的集成测试脚本。
5. 审核详情弹层和日志统计图。
6. Knife4j UI 或接口分组增强。
