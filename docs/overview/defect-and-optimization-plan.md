- 重做静态管理台视觉和信息架构，补齐审核中心、操作日志页面。

- 增加 RBAC 权限模型，支持管理员、角色、权限点、用户角色、角色权限和接口注解拦截。
- 增加过期上传会话定时清理，自动删除 `.chunks` 临时分片目录。
- 增加审核通过和上架前的安装包安全状态拦截。
- 增加 admin-service Dockerfile 和 Compose app profile，支持后端一键容器化启动。
- 测试覆盖持续补充，已覆盖上传、审核、权限、日志和安全拦截等核心路径。

- 抽离 `PackagePreparationService`，统一安装包规格校验、直传/分片会话互斥校验和 `AppPackageEntity` 构建，并补充直接单元测试。
- 修复空安全状态请求默认安全、非法 Token 可能返回 500、删除配置管理员登录入口并统一数据库身份来源、审核分配覆盖和重复活动审核任务竞态。
- 增加软件、版本、安装包、上传会话和审核任务状态枚举，减少魔法数字分散。
- 增加分片上传幂等覆盖、完成合并中的状态保护、正式包失败清理和上传会话失败补偿。
- 增加软件/版本/安装包数据库唯一约束和常用查询索引，补齐上传会话合并中状态。
- 分类、标签、软件详情缓存统一调整为事务提交后失效，避免回滚导致缓存提前失效。
- 操作日志改为事务提交后事件记录，避免业务事务回滚后留下误导性审计日志。
- 上传请求补充 `appKey`、`versionCode`、`expectedSha256` 等参数校验，并增加 GitHub Actions CI。
- 修复版本级审核驳回错误修改整个软件状态的问题，并限制可提审版本状态。
- 修复审核任务可分配给无审核权限账号、已分配任务可被其他审核人处理的问题。
- 修复无安装包或非可用安装包仍可能通过发布安全检查的问题。
- 修复同大小错误分片被误判为成功重试的问题，重复分片改为原子覆盖。
- 上传分片使用会话行锁串行更新进度；过期清理改为条件状态更新，避免清理已恢复上传的目录。
- 重复上架/下架改为幂等返回，避免刷新原始发布时间或写入重复日志。
- 修复操作日志按日期查询遗漏最后一秒毫秒数据和反向时间范围未校验的问题。
- 提交后操作日志写入失败不再让已成功业务请求返回失败；生产强审计场景明确演进为事务 Outbox。
- 删除未实际使用的状态变更 DTO，并将数据库唯一键冲突统一映射为 409。
- 最近一次本地测试为 83 个测试、0 失败、1 个因无 Docker 跳过。

## 当前缺陷

### 1. 前端仍是轻量验证台

当前 `admin-ui` 已覆盖软件、审核、操作日志、权限管理、分类、标签和联调检查，适合本地演示和接口验证。它仍然是纯静态页面，复杂交互、图表和权限菜单还没有做。

建议：后续只做必要增强，例如审核详情弹层、日志统计图和更完整的错误提示，不引入重型前端工程。

### 2. 上传治理仍需生产化

当前已经支持普通 Multipart、分片上传、断点续传、合并校验、SHA256 校验、可选 RSA 验签、分片原子覆盖、会话行锁、完成并发保护、条件过期清理和失败落盘文件清理，能覆盖内部研发测试阶段的大文件上传链路。本地目录存储适合当前 VM 联调；如果后续生产化，再替换为 MinIO / S3。

建议：后续增加后台定期孤儿文件巡检、批次总大小限制，并预留对象存储迁移接口。

### 3. 权限管理仍可生产化

当前已有 RBAC 表、默认角色权限数据、接口权限注解拦截、管理员账号管理、角色管理和角色授权页面。

建议：继续增加登录失败次数限制、密码复杂度策略、菜单级权限和按钮级权限隐藏。

### 4. API 文档仍可继续增强

当前已接入 springdoc-openapi 和 Swagger UI，可以在线查看和调试接口。后续如果希望更贴近国内 Java 后台项目习惯，可以替换或补充 Knife4j UI。

### 5. 部署链路仍需生产化

当前支持 Docker Compose app profile 一键启动 MySQL、Redis 和后端，但还没有生产级健康检查、镜像版本策略和部署说明。

建议：增加 admin-service 健康检查、镜像 tag 规范和生产部署文档。

### 6. 强审计一致性仍需 Outbox

当前操作日志监听器在业务事务提交后执行，避免回滚业务留下误导日志，也避免日志库瞬时异常让已经提交的业务返回假失败。代价是日志写入异常时存在审计缺口，当前会记录错误日志。

建议：生产环境如果要求审计绝不丢失，应在业务事务内写 Outbox 表，再由后台任务重试投递，而不是把普通 `AFTER_COMMIT` 事件描述成强一致。

## 优先级建议

1. 生产级密码策略、登录失败限制和菜单级权限隐藏。
2. 后台定期孤儿文件巡检和真实安装包安全扫描执行器。
3. Docker 生产部署说明。
4. Outbox、日志写入告警和失败重试。
5. 更完整的 MySQL / Redis / 文件链路集成测试。
6. 审核详情弹层和日志统计图。
7. Knife4j UI 或接口分组增强。
