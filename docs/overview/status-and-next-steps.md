# 当前进度和下一步

## 当前结构

```text
java-software-admin-service/
  admin-service/          # 主 Spring Boot 后台服务
  admin-ui/               # 本地验证用静态页面
  database/mysql/         # MySQL 初始化脚本
  docs/                   # 项目和面试文档
  docker-compose.yml      # MySQL + Redis + 可选后端服务
  Makefile                # 本地命令
```

## 已完成能力

后端：

- Spring Boot 应用骨架。
- 统一响应、错误码和全局异常处理。
- 管理员登录、Token 生成和鉴权拦截。
- RBAC 权限模型：管理员、角色、权限点、用户角色、角色权限、接口注解拦截和权限管理页面。
- 分类管理：新增、查询、树形、启停、删除校验。
- 标签管理：新增、查询、热门标记、删除校验。
- 软件管理：列表、详情、上传、编辑、上架、下架。
- 版本管理：新增版本、版本列表、最新版本控制。
- 安装包管理：新增 OS/CPU 架构/包格式变体和安装包安全状态管理。
- 代码结构：软件响应组装已从 `SoftwareServiceImpl` 抽离到 `SoftwareAssembler`，安装包规格校验、直传/分片会话二选一和安装包实体组装已抽离到 `PackagePreparationService`，核心 Service 更聚焦业务流程、状态校验和事务编排。
- 大文件上传：上传会话、分片上传、断点续传、分片原子覆盖、会话行锁、完成合并和业务消费。
- 上传治理：过期会话使用带时间条件的状态更新，确认仍过期后才在事务提交后删除分片，失败时清理已落盘正式包。
- 发布安全：审核通过和上架前拦截签名失败、未标记安全、处理失败或安全状态有风险的安装包。
- 审核并发控制：审核通过/驳回使用条件更新，首次分配使用待审核且未分配条件，活动审核任务通过数据库唯一键防止重复提审；已分配任务仅允许归属审核人处理。
- 审核流程：提交、分配、通过、驳回和历史记录；分配目标必须是启用且同时具备通过/驳回权限的管理员，版本驳回不再污染已上架软件状态。
- 操作审计：业务动作在事务提交后写入日志，失败只记录错误而不让已提交请求返回假失败；支持列表、详情、选项、统计和完整日期边界查询。
- 基础安全：Token 鉴权、Token 会话版本、非法 Token 统一返回 401、登录和 Token 验证统一使用数据库管理员、BCrypt 密码哈希、上传大小限制、安装包格式校验、安装包 SHA256 完整性校验、可选 RSA 签名验签。
- OpenAPI / Swagger UI：自动生成接口文档，支持在线调试。
- 本地冒烟脚本：覆盖登录、接口文档和核心查询接口。
- MyBatis XML Mapper。
- Redis 缓存：分类、标签、软件详情，写操作事务提交后统一失效。
- 独立 MySQL schema。
- Docker 后端镜像和 Compose app profile，一条命令可启动 MySQL、Redis 和后端。

前端：

- 管理后台布局。
- 软件列表、详情、上传、上架、下架。
- 审核中心：提交审核、分配审核人、通过和驳回。
- 操作日志：按动作、对象类型、对象编号和关键词查询。

## 已知不足

- 静态 UI 已覆盖软件、审核、操作日志、权限管理、分类、标签和联调检查，但仍是轻量管理台，不追求复杂前端工程化。
- 上传支持普通 Multipart 和分片上传，本地目录存储适配内部 VM 测试场景，已经有大小、格式、SHA256、可选 RSA 验签、分片原子覆盖、会话行锁、完成并发保护、条件过期清理和失败落盘文件清理。当前未接真实安全扫描执行器，安全状态由后台接口标记并进入审核/上架拦截。
- 当前已具备管理员账号管理、角色管理和角色授权页面。
- 对象存储不是当前内部研发测试主线；如果后续生产化，可将本地目录存储替换为 MinIO / S3。
- Swagger UI 已可用，暂未额外接入 Knife4j 皮肤。

## 面试主线建议

讲项目时不要从技术名词开始，先讲业务闭环：

> 这个项目是企业软件商店后台，核心要解决的是软件包从上传、审核到上架的管理链路。我的设计重点是把软件、版本、安装包、审核任务和操作审计串成一个事务清晰、状态可控、可测试的后台系统。

然后再讲技术点：

- Spring Boot 分层设计。
- MyBatis 数据访问和复杂查询。
- MySQL 事务保证多表一致性。
- Redis 缓存和缓存失效。
- 审核状态机。
- 操作审计。
- RBAC 权限模型。
- 上传安全、分片续传、签名校验和临时文件清理。
- Swagger 接口文档。
- 单元测试覆盖业务规则。
